                         🐛 BUGS ENCONTRADOS Y SOLUCIONADOS
═══════════════════════════════════════════════════════════════════════

1. ❌ → ✅ BUG CRÍTICO: IDs duplicados por race condition
   Función: POST /api/clientes (línea 67)
   Problema: countDocuments() + 1 → dos requests simultáneos = mismo ID
   Solución: Usar ObjectId automático de MongoDB (único + atómico)
   Impacto: Previene corrupción de BD en concurrencia

2. ❌ → ✅ BUG ALTO: Sin validación en POST
   Función: POST /api/clientes (línea 64-77)
   Problema: Aceptaba datos inválidos (DNI "papapapa", email sin @)
   Solución: Función validarCliente() valida nombre, dni, email, teléfono
   Impacto: Datos limpios garantizados

3. ❌ → ✅ BUG ALTO: GET by ID incompleto
   Función: GET /api/clientes/:id (línea 53-61)
   Problema: Solo buscaba por campo 'id', no por _id ni DNI
   Solución: Busca por ObjectId O por DNI (flexible)
   Impacto: GET funciona desde cualquier punto del sistema

4. ❌ → ✅ BUG BAJO: DELETE sin confirmación
   Función: DELETE /api/clientes/:id (línea 101-108)
   Problema: Retornaba success: true aunque cliente no existiera
   Solución: Retorna deletedCount y error 404 si no existe
   Impacto: UX clara (frontend sabe si realmente se eliminó)

5. ❌ → ✅ BUG ALTO: DNI no tiene validación UNIQUE
   Función: POST /api/clientes + BD indexes (línea 47)
   Problema: Permitía dos clientes con mismo DNI
   Solución: Índice unique: true + validación previa + error 409
   Impacto: Integración DECOLECTA funciona correctamente

                            📊 CAMBIOS POR ENDPOINT
═══════════════════════════════════════════════════════════════════════

POST /api/clientes
  ✅ Valida: nombre (requerido, string no vacío)
  ✅ Valida: dni (requerido, 8 dígitos)
  ✅ Valida: email (formato válido si se proporciona)
  ✅ Valida: teléfono (mín 7 dígitos si se proporciona)
  ✅ Valida: estado (activo, inactivo, suspendido)
  ✅ Previene: DNI duplicados (error 409)
  ✅ Responde: 201 + _id de MongoDB

GET /api/clientes/:id
  ✅ Busca por: ObjectId de MongoDB
  ✅ Busca por: DNI (8 dígitos)
  ✅ Retorna: 404 si no existe

PUT /api/clientes/:id
  ✅ Valida: mismas reglas que POST
  ✅ Busca por: ObjectId O DNI
  ✅ Evita: DNI duplicados (error 409)

DELETE /api/clientes/:id
  ✅ Retorna: deletedCount (0 si no existía)
  ✅ Retorna: 404 + error si cliente no existe
  ✅ Retorna: 200 + message si se eliminó

POST /api/equipos
  ✅ Valida: cliente_id (requerido)
  ✅ Valida: tipo_equipo (requerido)
  ✅ Previene: numero_serie duplicados (error 409)

POST /api/servicios
  ✅ Valida: nombre_servicio (requerido)
  ✅ Valida: categoria (requerido)
  ✅ Valida: costo_base (requerido)
